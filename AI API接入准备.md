三、API调用规范梳理

3.1 调用格式规范

请求方式
统一采用POST请求：所有AI API调用均使用POST方法
Content-Type：application/json;charset=UTF-8

请求参数格式
{
  "requestId": "唯一请求ID",
  "timestamp": "时间戳",
  "data": {
    // 业务数据
  },
  "signature": "签名"
}

响应格式
{
  "code": 200,
  "message": "success",
  "data": {
    // API返回结果
  },
  "requestId": "请求ID",
  "timestamp": "响应时间戳"
}

3.2 安全规范

接口签名加密
签名算法：HMAC-SHA256
签名字段：requestId + timestamp + data + secretKey
密钥管理：AccessKey存储在配置中心，不硬编码在代码中

数据脱敏规则
用户信息：手机号、姓名等敏感信息进行脱敏处理
资产信息：序列号、资产编号等关键信息保留，但不在日志中记录
传输加密：所有API调用必须使用HTTPS协议

3.3 异常处理方案

异常类型及处理策略
异常类型   HTTP状态码   处理策略
参数错误   400   返回具体错误信息，前端提示用户修正

认证失败   401   检查AccessKey配置，重新申请密钥

权限不足   403   检查API权限配置，联系管理员授权

请求超时   408   启动重试机制（最多3次），记录日志

服务不可用   500/503   启动降级策略，切换到人工模式

调用频率超限   429   暂停调用，等待冷却时间后重试

重试机制
重试次数：最多3次
重试间隔：指数退避（1s, 2s, 4s）
重试条件：仅对5xx错误和超时错误进行重试

各API调用限制
