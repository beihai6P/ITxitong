
一、功能清单

1.1 IT故障报修工单系统模块
功能名称   功能描述   对应角色   适配终端
自主报修   选择故障类型，填写问题描述，上传附件，选择紧急程度；提交时AI分析描述并推荐解决方案   普通员工   Web端、PC客户端、小程序端

工单跟踪   实时查看工单状态（待处理、处理中、已完成、已关闭），查看维修进度和维修人员信息   普通员工   Web端、PC客户端、小程序端

评价打分   维修完成后进行满意度评价（星级+文字备注），提出改进建议   普通员工   Web端、PC客户端、小程序端

接单模式   支持自动派单（按维修员负责区域/部门）和手动抢单双模式；结合AI预测结果优先派单   运维维修员   Web端、PC客户端

流程管控   遵循"待接单→维修中→待验收→已完成→驳回/关闭"流程，每步操作留痕可追溯   运维维修员   Web端、PC客户端

维修记录   填写故障原因、解决方法、更换配件、工时记录，上传维修后图片，形成完整维修档案   运维维修员   Web端、PC客户端

超时预警   设置工单响应超时、处理超时阈值，超时后通过弹窗、消息推送提醒相关人员   运维维修员、管理员   Web端、PC客户端、小程序端

AI辅助   接收AI预测的潜在故障提醒，获取AI推荐的维修方案、耗材准备建议   运维维修员   Web端、PC客户端

工单配置   自定义故障类型、工单流程节点、超时阈值、紧急程度分级；配置AI推荐触发条件   管理员   Web端

工单管理   查看所有工单详情、干预工单分配、处理异常工单、导出工单数据   管理员   Web端

统计分析   查看工单量、响应时长、完成率等数据，分析运维效率瓶颈   管理员   Web端

功能名称   功能描述   对应角色   适配终端

工作量统计   自动统计维修员接单量、完成率、响应时长、好评率，生成个人业绩报表   管理员、运维维修员   Web端

1.3 IT资产管理模块（含二维码集成）
功能名称   功能描述   对应角色   适配终端

离线盘点   支持运维人员在无网络环境下扫码盘点，网络恢复后同步盘点数据   运维维修员   PC客户端

差异对比   盘点完成后自动对比盘点数据与台账数据，生成差异报表   管理员、运维维修员   Web端、PC客户端

盘点记录   存档每次盘点详情、盘点人员、盘点时间，便于后续审计追溯   系统自动   全端

二、操作流程

2.1 报修流程

graph TD
    A[员工选择故障类型] --> B[填写问题描述]
    B --> C[上传故障现场图片/视频附件]
    C --> D[选择紧急程度]
    D --> E[系统通过AI API分析问题描述]
    E --> F{AI是否推荐解决方案?}
    F -- 是 --> G[展示AI推荐的解决方案]
    F -- 否 --> H[直接提交报修]
    G --> I{员工是否采纳AI建议?}
    I -- 是 --> J[员工自助解决问题]
    I -- 否 --> H
    H --> K[提交报修工单]
    K --> L[系统自动派单或开放抢单]
    L --> M[维修员接收工单]

2.2 AI推荐流程

graph TD
    A[员工提交报修工单] --> B[系统提取故障描述、资产类型、故障场景]
    B --> C[调用AI NLP API分析语义]
    C --> D[调用AI推荐算法API]
    D --> E[从问题库匹配最优解决方案]
    E --> F[AI生成补充建议]
    F --> G[返回推荐结果给前端]
    G --> H[前端展示AI推荐解决方案]
    H --> I[员工选择采纳或忽略]

2.3 资产盘点流程

graph TD
    A[管理员发起盘点任务] --> B[运维员通过PC客户端开始盘点]
    B --> C{是否有网络连接?}
    C -- 有 --> D[实时扫码盘点]
    C -- 无 --> E[离线扫码盘点]
    E --> F[网络恢复后同步数据]
    D --> G[系统实时验证资产信息]
    G --> H[标记盘盈/盘亏资产]
    H --> I[生成盘点差异报表]
    I --> J[管理员审核盘点结果]
    J --> K[更新资产台账]

2.4 大屏数据筛选流程

graph TD
    A[管理员进入数据大屏] --> B[选择筛选条件]
    B --> C[选择时间范围]
    C --> D[选择部门]
    D --> E[选择资产类型]
    E --> F[选择故障类型]
    F --> G[点击应用筛选]
    G --> H[大屏实时刷新数据]
    H --> I{需要查看详细数据?}
    I -- 是 --> J[点击图表钻取详情]

四、交互逻辑

4.1 AI功能交互逻辑

AI推荐结果展示
触发时机：员工提交报修表单时
展示位置：报修表单下方弹出卡片式推荐区域
展示样式：
  标题："AI智能推荐解决方案"
  内容：匹配的问题库解决方案 + AI生成的补充建议
  操作按钮："采纳建议"（关闭报修）、"仍需报修"（继续提交）
异常处理：
  AI API调用失败：显示"暂无推荐方案，请直接提交报修"
  无匹配方案：显示"未找到相关解决方案，请直接提交报修"

AI预测提醒展示
触发时机：维修员登录系统或进入工单页面时
展示位置：PC客户端右下角弹窗、Web端顶部通知栏
展示样式：
  标题："AI预测提醒"
  内容：预测的故障类型、高发区域、建议检查项
  操作按钮："查看详情"、"稍后处理"
异常处理：
  无预测数据：不显示提醒
  AI API异常：显示"AI服务暂时不可用"

4.2 数据大屏交互逻辑

数据筛选交互
筛选控件位置：大屏顶部固定区域
筛选逻辑：
  多条件组合筛选，实时生效
  选择条件后立即刷新相关图表数据
  清空按钮重置所有筛选条件
视觉反馈：
  已选条件高亮显示
  数据加载时显示loading动画
  筛选结果为空时显示"暂无数据"

详情钻取交互
触发方式：点击图表中的数据点、柱状图、饼图区块
钻取效果：
  平滑过渡到详情页面
  保留筛选条件上下文
  详情页面包含返回按钮
钻取层级：
  第一层：汇总数据（如部门工单量）
  第二层：明细数据（如具体工单列表）
  第三层：单条记录详情

大屏自定义配置
配置入口：大屏右上角"设置"按钮
配置项：
  模块开关：勾选需要展示的模块
  图表类型：为每个模块选择合适的图表
  刷新频率：设置数据自动刷新间隔
保存逻辑：
  配置实时预览
  点击"保存"后持久化配置
  支持"重置为默认"操作

4.3 通用交互逻辑

按钮点击反馈
成功操作：绿色toast提示"操作成功"，2秒后自动消失
失败操作：红色toast提示具体错误信息，3秒后自动消失
加载状态：按钮显示loading动画，防止重复点击
禁用状态：灰色按钮，hover显示禁用原因

表单验证反馈
实时验证：输入时实时校验，错误时显示红色边框
提交验证：提交时统一校验，错误字段滚动到可视区域
错误提示：字段下方显示具体错误信息
成功状态：正确输入显示绿色对勾图标

页面跳转逻辑
路由守卫：未登录用户跳转到登录页
权限控制：无权限页面显示"暂无权限"提示
返回逻辑：详情页返回列表页保持筛选状态
新开窗口：外部链接在新标签页打开
