
一、业务需求分析与对齐

1.1 核心业务场景梳理
业务模块   核心用户角色   关键业务流程   数据流转特点
工单管理   终端用户、运维工程师、管理员   提报→分配→处理→验收→归档   高频读写、状态流转复杂、需追溯完整生命周期

资产管理   资产管理员、运维工程师   入库→领用→调拨→报废   数据变更频率低但准确性要求高、需版本追溯

AI功能   运维工程师、系统管理员   问题识别→智能推荐→效果反馈   实时性要求高、需记录推荐效果用于模型优化

数据大屏   管理层、运维主管   数据采集→聚合→可视化展示   读多写少、需支持多维度聚合查询

1.2 用户需求优先级评估

高优先级需求（MVP必备）：
✅ 工单全流程状态追踪与通知
✅ 资产基础信息管理与二维码绑定
✅ 用户权限精细化控制
✅ 基础数据统计与报表

中优先级需求（V1.1迭代）：
⚡️ 知识库智能推荐与学习
⚡️ 工单自动分配与负载均衡
⚡️ 资产变更历史追溯

低优先级需求（远期规划）：
🔄 跨系统数据集成
🔄 预测性维护与预警

二、数据库设计评审建议

2.1 表结构设计优化建议

基础支撑类表优化
用户表（user）
建议增加：last_login_time（最后登录时间）、account_status（账户状态）
业务价值：支持登录行为分析、账户安全管控

部门表（department）
建议增加：parent_id（父部门）、sort_order（排序）
业务价值：支持组织架构树形展示、部门权限继承

核心业务类表优化
工单表（work_order）
建议增加：priority_level（优先级）、sla_deadline（SLA截止时间）、auto_assign_rule（自动分配规则）
业务价值：支撑智能工单分配、超时预警机制

工单流转日志表（work_order_flow_log）
建议新增：记录每次状态变更的详细信息
业务价值：完整追溯工单生命周期、支持流程优化分析

AI相关类表优化
AI推荐记录表（ai_recommendation_log）
建议增加：user_feedback（用户反馈）、recommendation_accuracy（推荐准确率）
业务价值：构建推荐效果评估闭环、持续优化AI模型

AI预测结果表（ai_prediction_result）
建议增加：confidence_score（置信度）、prediction_type（预测类型）
业务价值：支持预测结果可信度评估、多场景预测管理

2.2 关联关系设计关键点
关联场景   推荐方案   业务考量
工单与资产关联   工单表增加asset_id外键   支持资产故障快速定位、维修历史追溯

工单与知识库关联   工单处理记录表关联solution_id   积累解决方案使用效果、优化知识库推荐

用户与部门关联   用户表关联department_id   支持部门级数据权限隔离、组织架构管理

资产与二维码关联   二维码信息表关联asset_id   支持扫码快速查询资产信息、盘点效率提升

2.3 性能优化策略

高频查询场景索引设计
工单查询场景：
CREATE INDEX idx_wo_status_creator ON work_order(status, creator_id, create_time)
CREATE INDEX idx_wo_assignee_status ON work_order(assignee_id, status)
业务价值：支持运维工程师快速查看待处理工单

资产查询场景：
CREATE INDEX idx_asset_code_type ON asset(asset_code, asset_type)
CREATE INDEX idx_asset_status_location ON asset(status, location_id)
业务价值：支持资产快速定位、库存盘点效率提升

数据归档策略
工单数据：超过1年且状态为"已完成"的工单建议归档至历史表
操作日志：超过6个月的操作日志建议归档或删除
业务价值：保持主表数据量可控、查询性能稳定
